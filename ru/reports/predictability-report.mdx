## Обзор

Predictability Chart (Диаграмма предсказуемости) - это метрика, предложенная Kanban University для измерения стабильности и предсказуемости потока работы команды. Этот отчет помогает оценить, насколько надежно команда может прогнозировать время выполнения задач.

## Построение Predictability Chart

### 1. Сбор данных

- Собираются данные о lead time или cycle time для каждой задачи за определенный период.
- Важно обеспечить точность данных и охват достаточного промежутка времени для статистической значимости.

### 2. Расчет процентилей

- **50-й процентиль (медиана)**: Значение времени, ниже которого выполняется 50% задач.
- **98-й процентиль**: Значение времени, ниже которого выполняется 98% задач.

### 3. Вычисление отношения 98% / 50%

- Отношение = 98-й процентиль / 50-й процентиль
- Это отношение показывает степень вариативности в процессе. Меньшее значение означает более предсказуемый процесс.

### 4. Построение графика

- **Ось X**: Временная шкала (дни, недели или месяцы)
- **Ось Y**: Рассчитанное отношение 98% / 50% за каждый период
- Точки на графике соединяются линией для визуализации трендов

## Интерпретация значений

- **Значение близкое к 1**: Указывает на высокую предсказуемость и низкую вариативность процесса.
- **Значения от 1 до 2**: Считаются очень хорошими показателями.
- **Значения от 2 до 3**: Указывают на умеренную предсказуемость.
- **Значения выше 3**: Сигнализируют о низкой предсказуемости и высокой вариативности в процессе.

## Анализ трендов

- **Снижение отношения со временем**: Указывает на улучшение процессов и повышение стабильности.
- **Увеличение отношения**: Может сигнализировать о возможных проблемах или изменениях в процессе, требующих внимания.
- **Стабильная линия**: Говорит о неизменном, но не улучшающемся процессе.

## Настройка отчета

Под диаграммой расположены элементы управления:

1. **Resolution**: Выбор временного интервала (день, неделя, месяц).
2. **Select Swimlanes**: Фильтрация по конкретным свимлейнам.
3. **Select Columns**: Выбор колонок для анализа.
4. **Filtering Mode**: Выбор режима фильтрации (Activity/Columns).
5. **Completion Columns**: Выбор колонок завершения (для режима Columns).

## Практические советы по использованию

1. **Регулярный мониторинг**: Обновляйте и анализируйте график регулярно для своевременного обнаружения отклонений.

2. **Поиск причин вариативности**: При высоком отношении анализируйте задачи с длительным временем выполнения для выявления причин задержек.

3. **Внедрение улучшений**: Используйте полученные данные для оптимизации процессов, устранения узких мест и повышения эффективности.

4. **Комплексный анализ**: Рассматривайте Predictability Chart в сочетании с другими метриками, такими как Cycle Time и Throughput, для полноценного анализа.

5. **Учет контекста**: Принимайте во внимание специфику проекта и команды при интерпретации значений.

## Ограничения метрики

- Не учитывает размер или сложность задач.
- Может быть чувствительна к выбросам, особенно при небольшом количестве задач.
- Не отражает причины изменения предсказуемости.

## Заключение

Predictability Chart - ценный инструмент для оценки стабильности и предсказуемости процесса разработки. Регулярный анализ этой метрики поможет вам выявлять области для улучшения, оптимизировать рабочие процессы и повышать эффективность команды.
