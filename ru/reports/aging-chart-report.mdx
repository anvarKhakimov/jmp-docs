## Обзор

Aging Chart (Диаграмма старения) - это визуальный инструмент для анализа времени выполнения задач, которые в данный момент находятся в процессе работы. Этот отчет помогает быстро оценить распределение задач по времени и выявить потенциальные проблемы или узкие места в процессе поставки.

## Структура диаграммы

[Изображение Aging Chart]

- **Ось X**: Колонки доски Jira
- **Ось Y**: Время, которое задачи накопили в выбранных колонках (в днях)

Каждая задача представлена точкой на диаграмме:
- Чем выше точка, тем дольше задача находится в работе.
- Если несколько задач имеют примерно одинаковое время выполнения, они группируются в общую точку с указанием их количества.

## Интерактивные элементы

При наведении курсора на точку появляется подсказка со следующей информацией:
- Ссылка на задачу в Jira
- Накопленное время в выбранных колонках

[Изображение подсказки при наведении]

## Расчет статистики (процентилей)

Статистика рассчитывается на основе всех задач, попавших в выборку, включая исторические данные. Учитываются применяемые фильтры, выбранные даты, свимлейны и колонки.

Пользователи могут выбрать способ расчета статистики:
- **All columns**: Учитывается время во всех выбранных колонках.
- **Last Column**: Учитывается время во всех выбранных колонках, кроме последней (обычно колонка "Готово").

## Цветовое кодирование

На основе рассчитанных процентилей настраивается цветовое кодирование задач:

[Изображение цветового кодирования]

При выборе одной процентили (например, 50%):
- **Зеленый**: Задачи, накопившие времени меньше, чем 50% процентиль.
- **Красный**: Задачи, накопившие больше времени, чем 50% процентиль.

При выборе двух процентилей (например, 50% и 85%):
- **Зеленый**: Задачи, накопившие времени меньше, чем 50% процентиль.
- **Оранжевый**: Задачи между 50% и 85% процентилями.
- **Красный**: Задачи, накопившие больше времени, чем 85% процентиль.

Важно отметить, что высота цветных блоков различается для каждой колонки. Это связано с тем, что процентили рассчитываются отдельно для каждой колонки, учитывая накопленное время в предыдущих колонках.

## Дополнительные возможности

- **Show calculation details**: Позволяет увидеть сырые данные, на основе которых производился расчет.

## Интерпретация результатов

1. **Выявление аномалий**: Задачи, находящиеся значительно выше остальных, могут требовать особого внимания.

2. **Анализ распределения**: Оцените, как распределены задачи по времени в различных колонках. Это может указать на узкие места в процессе.

3. **Раннее обнаружение проблем**: Обратите внимание на задачи, которые находятся в красной зоне уже в начальных колонках. Это может сигнализировать о потенциальных проблемах на ранних этапах.

4. **Оценка эффективности процесса**: Большое количество задач в верхней части диаграммы может указывать на необходимость оптимизации процесса.

## Рекомендации по использованию

1. **Регулярный мониторинг**: Используйте Aging Chart для ежедневного или еженедельного обзора текущего состояния задач.

2. **Настройка процентилей**: Экспериментируйте с различными значениями процентилей для лучшего понимания распределения задач.

3. **Сочетание с другими метриками**: Используйте Aging Chart вместе с другими отчетами, такими как Cycle Time, для получения полной картины эффективности процесса.

4. **Фокус на аномалиях**: Уделяйте особое внимание задачам, которые значительно выбиваются из общего распределения.

## Заключение

Aging Chart - мощный инструмент для визуализации текущего состояния задач и выявления потенциальных проблем в процессе работы. Правильное использование этого отчета поможет вам оперативно реагировать на отклонения и поддерживать эффективность вашего рабочего процесса.
