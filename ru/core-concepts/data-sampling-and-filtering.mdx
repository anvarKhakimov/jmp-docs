## Общий принцип выборки данных

Jira Metrics Plugin (JMP) использует единый подход к выборке данных для всех отчетов. Плагин получает настройки вашей доски и на их основе формирует выборку задач для анализа. По умолчанию JMP включает в выборку все задачи, которые когда-либо были на доске, во всех колонках и свимлейнах.

## Инструменты фильтрации

JMP предоставляет несколько инструментов для уточнения выборки:

### Select Swimlanes

Дает возможность выбрать конкретные свимлейны для анализа, что позволяет сфокусироваться на определенных группах задач.

### Quick Filters

JMP автоматически добавляет все быстрые фильтры с вашей доски, позволяя использовать их для дополнительной фильтрации задач. Например, вы можете использовать быстрый фильтр для отбора задач конкретной команды.

### Timeframe

Позволяет указать период времени для анализа с помощью полей "from" и "to". Это помогает сосредоточиться на конкретном временном отрезке.

### Select Columns

Используется для выбора колонок, по которым будет рассчитываться lead time. В режиме фильтрации "Activity" этот параметр также влияет на фильтрацию задач - отображаются только те, которые были активны в выбранных колонках.

### Filtering Mode

Определяет способ фильтрации задач. Доступны два режима: "Activity" и "Completion". Подробное описание этих режимов вы можете найти в статье [Режимы фильтрации](/ru/docs/core-concepts/filtering-modes).

### Completion Columns

Этот параметр активен только в режиме фильтрации "Completion". Он позволяет указать колонки, прохождение через которые считается завершением задачи.

## Как это работает вместе

1. JMP сначала получает все задачи с доски.
2. Применяет фильтрацию по выбранным свимлейнам (Select Swimlanes).
3. Применяет выбранные быстрые фильтры (Quick Filters).
4. Затем применяет фильтр по времени (Timeframe).
5. Применяет фильтрацию по колонкам в зависимости от выбранного режима фильтрации (Filtering Mode).
6. Если выбран режим "Columns", учитывает Completion Columns.

## Пример использования

Предположим, вы хотите проанализировать работу конкретной команды за последний месяц. Вот как вы можете использовать фильтры:

1. Выберите свимлейн, соответствующий типу задач вашей команды.
2. Используйте быстрый фильтр "Team Alpha" (если у вас есть такой фильтр на доске).
3. Установите Timeframe на последний месяц.
4. В Select Columns выберите колонки, время нахождения задач в которых вы хотите проанализировать. Это определит, какие этапы рабочего процесса будут учитываться при расчете метрик.
5. Выберите подходящий режим фильтрации (Activity или Columns).

## Рекомендации по использованию

- Начните с широкой выборки, постепенно сужая её с помощью фильтров для более детального анализа.
- Экспериментируйте с разными комбинациями фильтров, чтобы получить наиболее полезные для вас данные.
- Используйте быстрые фильтры для быстрого переключения между различными наборами задач, например, между разными командами или типами работ.
- Помните, что выбор режима фильтрации может существенно повлиять на результаты анализа. Убедитесь, что вы понимаете разницу между режимами "Activity" и "Columns".

Правильное использование фильтров позволит вам получить именно те данные, которые необходимы для эффективного анализа и улучшения ваших рабочих процессов.
